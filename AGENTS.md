# AGENTS.md

## 项目定位

Stack Mentor AI 是面向前端开发者的 AI 框架源码学习工具。核心目标是帮助用户理解 JavaScript、TypeScript、Vue3、Vite 等技术的 API 用法、设计思想、底层实现、源码执行流程和工程实践。

所有实现都必须服务于一个原则：

> 不仅告诉用户“怎么用”，更要告诉用户“为什么这样设计”。

## 当前阶段

当前优先级是 MVP 第一阶段：

1. Vue3 源码知识库
2. AI 问答
3. API 解释器
4. 源码定位
5. 调用链分析

不要在没有明确需求时提前扩展 React、Node、企业版、多租户、支付、课程系统等后续阶段能力。

## 技术约束

- 包管理必须使用 PNPM，不使用 npm、yarn 或 bun 管理依赖。
- 项目必须采用 monorepo 架构组织，优先使用 `pnpm-workspace.yaml` 管理 workspace。
- 推荐目录结构：`apps/web` 放前端应用，`apps/api` 放后端服务，`packages/*` 放共享能力，例如 `ui`、`hooks`、`prompts`、`rag`、`types`。
- 前端目标技术栈：Vue3、Vite、TypeScript、PrimeVue、TailwindCSS v4+。
- Vue3 与 Vite 必须使用最新稳定版本；Vite 版本要求为 `8+`。
- 前端 UI 组件库必须使用 PrimeVue，不使用 Ant Design Vue、Element Plus、Naive UI 等其他组件库。
- CSS 工具链必须使用 TailwindCSS v4+。
- 页面动画效果必须使用 Vue Bits，常规交互动效优先复用 Vue Bits 能力。
- 编辑器优先使用 Monaco Editor。
- 图谱优先使用 Vue Flow 或 AntV G6。
- 后端使用 Node.js + NestJS，并必须使用 Fastify 适配器；Spring Boot 只作为备选方案。
- LLM 接入必须通过统一 Provider 接口封装，不要把单个模型供应商逻辑散落在业务代码里。
- RAG 数据源必须保留来源元数据，包括文档 URL、仓库、文件路径、版本、commit、标题、符号名。

## AI 回答约束

- 默认使用中文。
- 回答必须尽量基于官方文档、RFC、源码、Issue、PR 等可信资料。
- 不能确认的源码路径、函数名、设计动机必须标注为“不确定”或“需要检索确认”。
- 对 API 的解释必须包含：作用、参数、返回值、使用场景、常见错误、性能注意事项、设计动机、源码位置。
- 对源码问题必须包含：源码入口、关键函数/类、调用链、阅读顺序、设计思想。
- 对调用链问题必须区分同步流程、异步调度、响应式触发、最终结果。
- 对面试题必须分初级、中级、高级答案，并给出追问和评分标准。

## RAG 约束

- 禁止只按固定长度粗暴切分源码。
- 源码 chunk 应优先按文件、导出符号、函数、类、注释块或调用关系组织。
- 文档、RFC、源码、Issue、PR 应分来源类型入库。
- 回答必须能展示引用来源。
- 检索证据不足时，必须明确说明不足，而不是补全不存在的事实。

## UI/UX 约束

- 页面应优先体现“学习工作台”而不是普通聊天窗口。
- 复杂回答要支持代码块、调用链、图谱、引用来源和分层解释。
- 不要让长文本撑破布局；代码、图谱、引用区域都要有明确滚动或折叠策略。
- 空状态、加载中、检索失败、模型超时、无证据等状态必须可见。
- 初级用户入口要降低理解成本，高级用户入口要支持源码路径、调用链和架构分析。

## 开发约束

- 优先围绕 MVP 纵向打通闭环，不做大而全的横向功能堆叠。
- 前端页面与业务逻辑必须保持独立：页面组件负责布局和组合，不直接承载复杂业务逻辑。
- 可复用业务逻辑必须抽成独立 hook 使用，推荐放在 `apps/web/src/hooks` 或共享包 `packages/hooks`。
- Hook 命名使用 `useXxx`，例如 `useApiExplainer`、`useSourceLocator`、`useCallChainAnalysis`。
- 页面、组件、hook、类型、请求服务要分层组织，避免一个文件同时处理 UI、状态、请求、数据转换和提示词拼接。
- 每个核心功能都要有可验证样例，例如 `watch()`、`ref()`、`computed()`、`count.value++`。
- 新增功能前先明确它属于 API 解释、源码阅读、调用链分析、知识图谱、执行流程、AI 实验室、面试官还是知识库问答。
- 涉及模型调用的功能必须有错误处理、超时处理、重试或降级提示。
- 不要把 prompt 硬编码在组件里，应集中管理或通过模板层封装。

## 验收标准

- 用户能输入一个 Vue3 API 并获得可学习、可溯源的解释。
- 用户能输入一个源码主题并获得源码入口和阅读路径。
- 用户能输入一个代码片段并看到调用链分析。
- 回答中事实、源码位置和推断边界清晰。
- MVP 功能能用最小成本验证产品价值。
